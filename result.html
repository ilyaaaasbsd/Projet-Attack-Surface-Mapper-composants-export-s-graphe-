📋 Recommandations Générales

🔒

Principe du moindre privilège

Chaque composant doit déclarer android:exported="false" sauf s'il doit être accessible par d'autres applications. Protégez les composants publics avec des permissions custom.

🛡️

Network Security Config

Créez un fichier res/xml/network_security_config.xml et référencez-le dans le Manifest pour contrôler finement les connexions réseau autorisées.

💾

Protection des données

Désactivez le backup ADB (allowBackup="false"), chiffrez les données locales sensibles et utilisez le Keystore Android pour les clés cryptographiques.

🔗

Deep Links & Intent Security

Validez rigoureusement tous les paramètres reçus via des intents ou des deep links. Ne faites jamais confiance aux données provenant de sources externes.

🧪

Tests de sécurité

Utilisez des outils comme MobSF, Drozer ou Frida pour tester dynamiquement votre application contre les vulnérabilités détectées.

📦

Build de production

Assurez-vous que android:debuggable est absent ou à "false" dans le build de production. Activez R8/ProGuard pour l'obfuscation du code.
